BASIC ACTIVATION FUNCTIONS

Overview

Activation Functions

Since neural networks evolved from simpler models like linear regression, they rely on linear layers to determine magnitude and direction. Activation functions introduce non-linearity by "breaking" the straight output of the linear layers to provide shape. By creating these pivot points, they allow the network to evolve beyond simple regression trends and learn complex features.

Sigmoid

σ(x) = 1 / (1 + e⁻ˣ)

Output range: [0, 1]

Historically used in output layers for binary classification (interpretable as probability).

Major drawbacks:

  • Vanishing gradient: gradients near 0 or 1 become tiny → slow or stalled learning in early layers.
  • Outputs are not zero-centered → can cause zig-zagging updates.
Tanh (Hyperbolic Tangent)

tanh(x) = (eˣ - e⁻ˣ) / (eˣ + e⁻ˣ)

Output range: [-1, 1]

Zero-centered (better than sigmoid for hidden layers) → gradients more likely to point in useful directions.

Still suffers from vanishing gradient problem for large |x|.

ReLU (Rectified Linear Unit)

f(x) = max(0, x)

Output range: [0, ∞)

As the network learns to shift irrelevant noise into the negative range, ReLU "switches off" those neurons. This resulting field of zeros creates sparsity, ensuring the model processes only meaningful signals.ReLU converts all negative values to zero.

Advantages:

  • Very fast to compute (simple thresholding).
  • No vanishing gradient for positive values → faster convergence in practice.
  • Promotes sparsity (many neurons output 0) → This makes the model efficient because it processes less data.
  • Sparsity is helpful because it treats irrelevant data as non-existent, rather than "opposite" (negative).
  • Sparsity works by having the linear layer learn to subtract a bias, shifting normal signals below zero. Only anomalies overcome this threshold to become positive and activate.
  • Without sparsity, every neuron contributes to every decision, creating a noisy blend. Sparsity forces neurons to be specialists, reducing overfitting.

Main issue: Dying ReLU problem — neurons can get stuck outputting 0 forever if they receive only negative inputs (gradient = 0).

Leaky ReLU

f(x) = max(αx, x) where α is a small constant (typically 0.01)

Output range: (-∞, ∞)

Fixes dying ReLU by allowing a small, non-zero gradient when x < 0 → keeps neurons "alive" and learning even when mostly inactive.

Variants: Parametric ReLU (PReLU — α is learned), ELU, GELU, Swish, etc., offer further improvements in some cases.

Quick Comparison Summary
  • Sigmoid: Good for binary output layers; avoid in hidden layers.
  • Tanh: Better than sigmoid for hidden layers; still prone to vanishing gradients.
  • ReLU: Default choice for most hidden layers — fast & effective.
  • Leaky ReLU / variants: Use when you suspect dying neurons or want smoother behavior on negatives.
